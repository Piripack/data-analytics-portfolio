Project 01

EventEase

A full-stack event booking system built with React, Express, SQLite, Material UI, session-based authentication, and Leaflet maps. Users can search events by location, view tickets, log in, and make bookings through a browser-based interface.

Back to module index Next: BookFinder

Overview

EventEase is the stronger of the two module projects because it combines a real frontend application with a modular Express backend and a populated SQLite database. The codebase includes server routing, session handling, client-side state management, interactive map rendering, booking workflows, and a recorded demo.

React Material UI Express SQLite express-session Leaflet

What Was Built

  • REST-style API endpoints for event search by location, ticket retrieval by event, booking creation, login, logout, and current-session checks.
  • A React frontend with a searchable event list, login modal, booking dialog, booking history view, and theme toggle.
  • Leaflet and OpenStreetMap integration for showing event locations on a map with booking actions triggered from map markers.
  • Session-based access control so bookings are tied to a logged-in user.
  • Error handling for missing booking fields, insufficient ticket availability, and authentication failures.

Technical Evidence

  • The backend is split into routers, controllers, DAO logic, and middleware rather than one monolithic file.
  • The booking controller validates required fields, blocks bookings for past events, checks ticket availability, updates availability, and inserts booking records.
  • The frontend uses React component composition across EventSearch, EventList, EventMap, BookTicketDialog, LoginForm, and MyBookings.
  • The SQLite database contains 5 events, 11 ticket rows, 30 bookings, and 4 users.

Business Value

The project demonstrates the kind of full-stack flow expected in a small production-style web app: search, browse, authenticate, transact, and review user-specific data. It shows more than just endpoint construction because the frontend and backend are clearly integrated around a real booking journey.
