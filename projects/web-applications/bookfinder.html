Project 02

BookFinder

Smaller Express and SQLite web application for searching books, logging in, reserving titles, and adding new books with role-based restrictions.

Back to module index Back to EventEase

Overview

BookFinder is a more constrained exercise than EventEase, but it still shows useful web application fundamentals: AJAX requests from the browser, Express routes, SQLite queries, session handling, reservation logic, and admin-only actions. The code is task-oriented because it was built from a time-constrained assessment template, but the completed functionality is still clear.

HTML Client-side JavaScript Express SQLite express-session

What Was Built

  • Book search by title and author using AJAX and SQL LIKE filters.
  • Book detail retrieval as JSON via an Express route.
  • Reservation handling with insert logic into a reservations table and book availability updates.
  • Session-based login handling on the server and login requests from the client.
  • Admin-gated add-book functionality with user-friendly error messaging on missing fields.

Technical Evidence

  • The server includes routes for /searchBook, /book/:id, /reserveBook, /addBook, and /login.
  • The client JavaScript handles search, reservation, add-book submission, and login via fetch-based AJAX calls.
  • The SQLite schema includes users, books, and reservations tables.
  • The bundled database currently contains 3 books, 2 reservations, and 3 users.
  • The implementation enforces login checks for reservations and admin checks for adding books.

Business Value

This project is useful as evidence of core web development fundamentals: browser-to-server communication, JSON handling, database interaction, session logic, and basic role-based access control. It is a smaller exercise than EventEase, but still credible as a portfolio subsection or secondary project.
