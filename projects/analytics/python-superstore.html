Project 02

Python EDA and Data Cleaning on Superstore

Audited and profiled the Superstore transactional dataset in Python to validate structure, inspect distributions, and document repeatable cleaning steps before any KPI reporting or dashboard work.

Back to index Next: Tableau Dashboard

Overview

This project moved the Superstore analysis into a code-based workflow using pandas, numpy, and scikit-learn utilities. The emphasis was on data quality checks, profiling, and descriptive analysis that could be rerun consistently.

Python pandas numpy SimpleImputer

Approach

  • Loaded the dataset with pd.read_csv() and inspected structure with head(), tail(), and info().
  • Checked dimensions, data types, and memory usage.
  • Used isnull().sum() and drop_duplicates() as quality checks.
  • Tested imputation strategies with SimpleImputer, even though the documented dataset state showed no null values.
  • Calculated descriptive statistics and grouped results by category.

Key Findings

  • The dataset contained 9,994 rows and 22 columns, all non-null, with a reported size of 1.7 MB.
  • Sales ranged from 0.44 to 22,638.48, indicating a wide spread in transaction values.
  • The reported sales standard deviation was 623.25 and variance was 38,434.45, confirming high variability.
  • The workflow showed a clear shift from one-off spreadsheet analysis to a more repeatable code-based process.

Decision Implications

  • The dataset was structurally sound enough to analyse directly, which means effort could move from repair work into interpretation and reporting.
  • The wide spread in sales values means headline averages should be treated cautiously. Any KPI pack built on this data should include spread, median, or segment-level breakdowns.
  • The main practical gain was workflow reliability: once the checks are scripted, the same validation can be rerun before dashboard refreshes or further modelling.
